# Voice Journal / Notes — Research

## Overview

The bot currently handles voice messages by transcribing them via Groq Whisper and then routing the text through the same command/intent pipeline as typed messages. There is no concept of a "note" or "journal entry" — everything is either a task command, brain dump (which extracts tasks/people/ideas), or chat. The user wants to be able to send a voice note that gets saved as a journal entry with both the raw transcript and a Claude-generated summary/bullet points for later review.

## Architecture

### Current Voice Flow

```
User sends voice message
  → handle_voice() [bot/handlers.py:348]
    → download audio bytes from Telegram
    → transcribe_voice(audio_bytes) [services/groq.py] — Groq Whisper
    → reply "I heard: {text}"
    → process_input(text, source="voice")
      → route_command(text) — fast path or Claude classify
        → command dispatch (done, add, list, dump, etc.)
        → OR chat fallback
```

### Current Capture Model

The `Capture` table exists but is only used to store brain dump extractions:
- `raw_text` — JSON string of extracted items (NOT the original transcript)
- `source` — always "brain_dump"
- `items_created` — always empty list (never populated)
- `processed` — always True

This model is underutilized and could be repurposed or a new model could be created for notes.

### What Needs to Change

1. **New `Note` model** — Store voice journal entries with raw transcript, summary, bullet points, and optional tags/topics
2. **New Claude function** — Summarize a transcript into structured notes (bullet points, key themes, action items if any)
3. **New intent: "note"** — Claude classifier recognizes "add a note" or "note:" prefix
4. **New command: `/note`** — Direct trigger for note capture
5. **Note retrieval** — `/notes` to list recent notes, search notes by keyword

## Data Flow

### Proposed Note Flow

```
User sends voice message starting with "note" or "add a note"
  → handle_voice() transcribes via Groq
  → process_input() → route_command()
    → fast path: "note" prefix matched
    → OR Claude classifies intent as "note"
  → _handle_note(transcript, update)
    → Claude summarizes transcript into:
      - title (short, auto-generated)
      - bullet_points (key points extracted)
      - action_items (any implicit tasks mentioned, optional)
    → INSERT into notes table:
      - raw_transcript = full Groq output
      - summary = Claude bullet points
      - title = Claude-generated title
      - source = "voice" | "text"
    → Reply to user with title + bullets
```

### Proposed Note Model

```python
class Note(Base):
    __tablename__ = "notes"
    id = Column(String, primary_key=True, default=new_id)
    title = Column(String, nullable=False)          # Auto-generated by Claude
    raw_transcript = Column(Text, nullable=False)    # Original transcription
    summary = Column(Text, nullable=False)           # Bullet-point summary
    tags = Column(JSON, default=list)                # Auto-extracted topics/themes
    source = Column(String, default="voice")         # "voice" or "text"
    created_at = Column(TIMESTAMP(timezone=True), default=utcnow)
```

Fields rationale:
- `title` — Short label so notes are scannable in a list ("Thoughts on pricing model", "Med spa scheduler architecture")
- `raw_transcript` — The full unedited transcript for reference
- `summary` — Claude's structured summary with bullet points
- `tags` — Auto-extracted topics for future filtering (e.g. ["taxes", "business", "music"])
- `source` — Whether it came from voice or typed text
- No `notion_id` for now — notes are personal/internal, not synced to Notion

## Key Functions/Components

### New Claude Function: `summarize_note`

- **Purpose:** Take a raw transcript and produce a structured note
- **Input:** Raw transcript string
- **Output:** JSON with title, summary (bullet points), tags
- **Location:** `services/claude.py`
- **System prompt:** Instruct Claude to:
  1. Generate a short descriptive title (5-10 words)
  2. Extract key points as bullet points
  3. Preserve the speaker's intent and tone
  4. Extract 1-5 topic tags
  5. Flag any implicit action items (but don't create tasks — that's what `/dump` is for)

### New Command Handlers

| Command | Handler | Action |
|---------|---------|--------|
| `note [text]` / `/note [text]` | `cmd_note` | Save text as a note with Claude summary |
| `notes` / `/notes` | `cmd_notes` | List recent notes (last 10, titles + dates) |
| `notes [keyword]` / `/notes [keyword]` | `cmd_notes` | Search notes by keyword |
| `note [N]` / `viewnote [N]` | `cmd_viewnote` | View full note by number from last list |

### Intent Classifier Update

Add to classify_intent:
- `{"intent": "note", "text": "..."}` — user wants to save a note/journal entry
- `{"intent": "notes"}` — user wants to list/search notes. Optional: `"search": "keyword"`

### Voice Handler Update

No changes needed to `handle_voice` — it already transcribes and sends to `process_input`. The note routing happens in `route_command` via prefix match or Claude classification.
